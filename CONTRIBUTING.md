# Contributing to upi_pro_sdk

Thank you for your interest in contributing! 🎉

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/upi_pro_sdk.git`
3. Install dependencies: `flutter pub get`
4. Create a branch: `git checkout -b feat/your-feature-name`

## Development Guidelines

- Follow the [Dart style guide](https://dart.dev/guides/language/effective-dart/style)
- Add `///` doc comments to all public API members
- Ensure `flutter analyze` passes with zero issues
- Add or update tests for every change in `test/`
- Update `CHANGELOG.md` under `[Unreleased]`

## Running Tests

```bash
flutter test
```

## Running the Analyzer

```bash
flutter analyze
```

## Submitting a PR

1. Ensure all tests pass
2. Update `CHANGELOG.md`
3. Push your branch and open a Pull Request against `main`
4. Fill out the PR template

## Reporting Bugs

Open an issue at: https://github.com/satyam184/upi_pro_sdk/issues

Please include:
- Flutter & Dart version (`flutter --version`)
- Device / OS
- Steps to reproduce
- Expected vs actual behaviour
